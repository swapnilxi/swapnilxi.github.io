Architecture in one glance

From raw signals to CEO clarity.

  1. 1. Data in → orders, fulfillment logs, dark store cameras (zone-level only), ledgers.
  2. 2. Metrics → runway, burn, contribution margin, delivery economics, discount impact.
  3. 3. CV signals → congestion, queues, packing delays (no faces, no employee monitoring).
  4. 4. RAG → Neon Postgres + pgvector, LangChain, Zep/zvec.
  5. 5. Agents → CFO, Ops, Strategy, Supervisor on LangGraph.
  6. 6. Surfaces → CFOBuddy chat + Pulse CEO dashboard on a FastAPI + PyTorch backend.

Tech stack

  • Frontend: Next.js (CFOBuddy chat, Pulse dashboard)
  • Backend: FastAPI, LangGraph, LangChain
  • ML: PyTorch for CV & anomaly detection
  • RAG: Neon Postgres + pgvector, Zep/zvec
  • LLMs: DeepSeek-first with open-source fallbacks

AI safety

  • No facial recognition, no employee monitoring
  • Only aggregated operational signals
  • CFO approval gates for sensitive thresholds
